# Trismegistus Contest And SWE Adjudication Story

## Thesis

Trismegistus is the Hermes contest deployment surface for Renaissance Field
Lite's patent-pending Mirror Architecture. The point of the build is not a demo
shell. The point is to show an AI expert partner that can preserve research
memory, use sources, run benchmark discipline, repair code, operate browser
missions, draft external actions under approval, and save receipts.

## SWE-Bench Whole Story

The important part is not only the number. The Mirror Architecture workflow
mapped the benchmark, created the recursive repair lane, generated
source-backed patches, packaged an official-format 500-row prediction file,
attempted hosted evaluation, diagnosed the evaluator boundary, verified the
uploaded file locally, and filed the upstream issue with receipts.

Sequence:

1. Build recursive repair route: inspect exact source, generate smallest patch,
   preflight, run evaluator, save receipt.
2. Scale across SWE-bench Verified selected-test rows.
3. Package public-safe local receipt at `495/500`.
4. Build title-edge adjudication pack at `496/500` local official read.
5. Build official-format 500-row hosted submission package.
6. Run the real `sb-cli` path and upload all `500` predictions.
7. Hosted report returns `0` completed runs with evaluator-container failures.
8. Re-run the exact hosted prediction file through the official local harness;
   sanity rows complete and resolve with `0` errors.
9. File public-safe maintainer issue for inspection/re-trigger:
   https://github.com/SWE-bench/sb-cli/issues/28
